# PWA 및 로컬 데이터 관리 전략 보고서

**프로젝트:** Stock Snowball
**일자:** 2025-05-22
**작성자:** GSD Research Agent

## 1. 개요
'Stock Snowball'은 사용자가 매일 소액을 투자하여 자산을 키워가는 과정을 시뮬레이션하고 기록하는 도구입니다. 금융 도구로서의 **신뢰성(Data Integrity)**, **접근성(Offline Availability)**, 그리고 **보안성(Security)**을 극대화하기 위해 PWA(Progressive Web App) 도입과 로컬 퍼스트(Local-first) 데이터 전략을 제안합니다.

---

## 2. PWA 도입 전략: Apple 스타일 사용자 경험 구현

Apple의 디자인 철학을 따르는 서비스로서, 웹에서도 네이티브 앱과 구별할 수 없는 부드러운 경험을 제공하는 것이 핵심입니다.

### 2.1. 오프라인 접근성 및 성능
- **Service Worker (Workbox)**: 핵심 UI 자산(JS, CSS, Fonts) 및 정적 이미지를 캐싱하여 네트워크 연결 없이도 즉시 앱이 구동되도록 합니다. `Stale-While-Revalidate` 전략을 사용하여 백그라운드에서 업데이트를 진행합니다.
- **App Shell Architecture**: 데이터 로딩 전에도 Apple 특유의 깔끔한 UI 프레임을 즉시 표시하여 인지 대기 시간을 최소화합니다.

### 2.2. Apple 맞춤형 설치 경험 (iOS/iPadOS)
- **Splash Screens**: iOS는 `manifest.json`의 스플래시 설정을 무시하므로, `apple-touch-startup-image` 메타 태그를 기기 해상도별로 제공해야 합니다. `pwa-asset-generator`를 사용하여 iPhone 16 시리즈 등 최신 기기에 대응합니다.
- **Dark Mode Splash**: 사용자의 시스템 설정에 맞춘 다크 모드용 스플래시 이미지를 제공하여 시각적 일관성을 유지합니다.
- **Home Screen Icons**: iOS 18의 틴트 아이콘 트렌드를 반영하여, 배경이 투명하거나 시스템 색상과 조화로운 180x180px 고해상도 아이콘을 적용합니다.

## 3. 로컬 데이터 저장 전략

금융 데이터는 구조화되어 있고 양이 많아질 수 있으므로 단순한 `LocalStorage`보다는 `IndexedDB`가 적합합니다.

### 3.1. 기술 스택 추천: RxDB (Reactive Database)
금융 PWA를 위한 최적의 선택으로 **RxDB**를 제안합니다.
- **Local-first**: 모든 읽기/쓰기가 로컬에서 즉시 발생하며, 네트워크는 동기화 용도로만 사용됩니다.
- **NoSQL & Schema**: JSON 스키마를 통해 데이터 구조를 엄격히 관리하여 복리 계산의 정확성을 보장합니다.
- **Reactivity**: 데이터 변경 시 UI(차트, 요약 지표)가 즉각적으로 업데이트됩니다.

### 3.2. 데이터 저장소 비교

| 항목 | LocalStorage | IndexedDB (RxDB) | 비고 |
| :--- | :--- | :--- | :--- |
| **용량** | ~5MB (매우 제한적) | 수백 MB ~ GB (충분함) | 거래 내역 누적 시 필수 |
| **데이터 타입** | 문자열만 가능 | 객체, 바이너리 가능 | 암호화 데이터 처리에 유리 |
| **성능** | 동기 (메인 스레드 차단) | 비동기 (성능 저하 없음) | 대량 데이터 조회 시 유리 |
| **쿼리** | 지원 안 함 | 인덱싱 및 복합 쿼리 지원 | 기간별 투자 요약에 필요 |

## 4. 보안 및 동기화 아키텍처

### 4.1. 보안 전략 (Security-first)
금융 데이터의 민감성을 고려하여 다중 보안 계층을 적용합니다.
- **필드 레벨 암호화 (Field-level Encryption)**: RxDB의 내장 기능을 사용하여 잔액, 계좌 번호 등 민감한 필드를 `Web Crypto API (AES-GCM)`로 암호화합니다.
- **비추출 키(Non-extractable Keys)**: 암호화 키를 생성할 때 추출 불가능 옵션을 설정하여, XSS 공격으로 스크립트가 탈취되더라도 실제 키 값을 읽을 수 없도록 방어합니다.
- **CSP (Content Security Policy)**: 엄격한 CSP 설정을 통해 악성 스크립트 실행을 원천 차단합니다.

### 4.2. 데이터 동기화 아키텍처
사용자가 여러 기기에서 동일한 데이터를 확인할 수 있도록 동기화 환경을 구축합니다.
- **Local-first Sync**: 클라이언트가 주 데이터 저장소 역할을 하며, 서버는 백업 및 동기화 피어(Peer) 역할을 수행합니다.
- **충돌 해결 (Conflict Resolution)**: 타임스탬프 기반의 LWW(Last-Write-Wins) 또는 결정론적 복제 프로토콜을 사용하여 데이터 정합성을 유지합니다.
- **Background Sync API**: 네트워크 불안정 시에도 거래 기록이 유실되지 않도록 연결 복구 후 백그라운드에서 자동으로 서버에 전송합니다.

## 5. 단계별 구현 제안
1.  **1단계**: IndexedDB + RxDB 기반의 로컬 저장소 구축 및 핵심 계산 엔진 연동.
2.  **2단계**: Web Crypto API를 활용한 데이터 암호화 및 보안 가드레일 적용.
3.  **3단계**: PWA 매니페스트 및 서비스 워커 설정을 통한 오프라인 모드 및 iOS 설치 최적화.
4.  **4단계**: (선택) 외부 서버 연동을 통한 기기간 데이터 동기화 기능 확장.

이 전략을 통해 'Stock Snowball'은 단순한 계산기를 넘어, 사용자가 안심하고 사용할 수 있는 견고한 금융 관리 도구로 성장할 것입니다.
