# Phase 4.4 Plan: Performance Optimization

**Goal**: 차트 렌더링 성능을 개선하고, 애플리케이션의 전체 번들 사이즈를 최적화하여 쾌적한 사용자 경험을 제공합니다.

## Must-Haves
- [ ] `SnowballChart` 렌더링 최적화 (불필요한 계산 제거 및 메모이제이션 강화)
- [ ] 번들 사이즈 분석 및 코드 스플리팅 적용
- [ ] 차트 인터랙션(Scrubbing) 시 프레임 드랍 방지

## Task List

### 4.4.1 Chart Rendering Optimization
- [ ] `SnowballChart.tsx`: `allPoints` 및 스케일 계산 로직을 `useMemo`로 확실히 보호
- [ ] `handleTooltip`: `find()` 대신 인덱스 기반 접근으로 변경 (모든 시나리오가 동일한 날짜 축을 가진다고 가정하거나 정렬된 상태 활용)
- [ ] `ParentSize` 내에서 발생하는 빈번한 렌더링을 `React.memo` 또는 `useDeferredValue`로 제어 검토
- [ ] SVG 패스 단순화 또는 포인트 샘플링 (데이터가 너무 많을 경우)

### 4.4.2 Bundle Size & Code Splitting
- [ ] `vite-bundle-visualizer` (또는 유사 도구)를 사용하여 번들 구성 분석
- [ ] `RxDB` 및 `framer-motion` 등 대용량 라이브러리의 트리 쉐킹 확인
- [ ] 메인 레이아웃과 설정 패널 등을 `React.lazy`를 통한 Dynamic Import로 분리
- [ ] `@visx` 패키지 중 중복되거나 불필요한 기능 제거

### 4.4.3 Interaction Smoothing
- [ ] 툴팁 애니메이션 최적화 (이동 시에는 애니메이션을 끄고 나타날 때만 적용하거나, `layout` prop 활용 지양)
- [ ] 스크러빙 시 차트 전체가 리렌더링되지 않도록 툴팁 레이어를 별도 컴포넌트로 분리 검토

## Success Criteria
1. 번들 사이즈(gzipped)가 현재 565kB에서 유의미하게 감소 (목표: 400kB 이하).
2. 차트 스크러빙 시 60fps에 가까운 부드러운 인터랙션 유지.
3. 초기 로딩 속도(LCP) 개선.
