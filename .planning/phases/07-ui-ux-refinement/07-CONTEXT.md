# Phase 7 Context: UI/UX & Financial Precision Refinement

이 문서는 사용자의 자산 입력 편의성 개선, 통화 전환 로직 고도화, 그리고 고급 설정 UI 개선에 대한 결정 사항을 기록합니다.

## 1. 입력 편의성 개선 (Big Number Helper)
- **표시 방식**: 자산 및 납입액 입력창 하단에 희미한 텍스트(Subtle Text)로 실시간 단위 변환 결과를 표시함.
- **단위 대응**:
  - **KRW**: '억/만' 단위 표기 (예: 1억 2,500만 원).
  - **USD**: 'Million/Billion' 단위 표기 (예: 1.25 Million $).

## 2. 통화 전환 및 환율 로직
- **자동 환산**: 원/달러 전환 시 현재 입력된 금액을 환율에 맞춰 자동 변환하여 입력 필드에 반영함.
- **환율 설정**: 고급 설정(Advanced Settings) 내에 사용자가 직접 환율을 입력할 수 있는 필드 제공. (기본값: 1,450원)
- **표기 정밀도**:
  - **KRW**: 소수점 없이 정수 부분만 표기.
  - **USD**: 소수점 2자리까지 표기.
  - **내부 연산**: 모든 연산은 소수점 3자리까지 정밀하게 수행.

## 3. 투자 기간 슬라이더 및 입력 UX
- **슬라이더 동작**:
  - 시각적 스케일은 **30년(100%)**으로 고정.
  - 30년 초과 값 입력 시 슬라이더 핸들은 100% 위치에 고정되나, 숫자 입력창을 통해 값은 정상 반영.
- **입력창 '0' 처리**:
  - 입력 시 기존의 '0' 값이 지워지지 않아 발생하는 불편함 해결.
  - 입력창이 비어있거나 '0'을 지운 상태에서도 내부적으로는 기본값(0)으로 처리하여 UX 유연성 확보.

## 4. UI 구성 요소 (Apple Style)
- **고급 설정 버튼**:
  - 상단 헤더 바(Header Bar) 우측에 **톱니바퀴(Gear) 아이콘**으로 배치.
  - 전체 너비 바 형태가 아닌, 정교한 아이콘 버튼 형태로 UX 최적화.
- **동적 레이블**:
  - 고급 설정에서 선택한 '납입 주기(일/주/월)'에 따라 입력 필드 레이블이 실시간 변경됨. (예: "주 납입액")
  - 기본값은 **'일(Daily)'**로 설정.

## 5. 지연/보류 사항 (Deferred)
- 복잡한 차트 타입 변경이나 대규모 데이터 마이그레이션은 본 페이즈 범위에서 제외.

## 6. 성공 기준 (Success Criteria)
- 사용자가 큰 금액 입력 시 즉각적으로 단위를 인지할 수 있음.
- 원/달러 전환 시 자산 시뮬레이션의 연속성이 유지됨.
- 투자 기간 입력이 30년을 넘어서도 직관적으로 동작함.
- 입력창의 숫자 수정이 막힘없이 부드럽게 작동함.
